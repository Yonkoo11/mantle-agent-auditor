How a verdict is made

  1. 01
    Read the code. A hybrid engine — Slither static analysis plus an LLM reasoning pass on Tencent Hunyuan — finds the bugs with line numbers.
  2. 02
    Anchor the report. The full report is hashed with keccak256, so the on-chain verdict is tamper-evident and falsifiable.
  3. 03
    Write the verdict. A pass/fail attestation is written to the registry on Mantle — any contract can call isAttestedSafe(addr).
  4. 04
    Bind to identity. If the target is an ERC-8004 agent, the verdict is written to its reputation too — safety travels with the agent.
