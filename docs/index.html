How it works

  1. Read the code. A hybrid engine (Slither static analysis + Tencent Hunyuan reasoning) finds the bugs with line numbers.
  2. Anchor the report. The full report is hashed (keccak256) so the on-chain verdict is tamper-evident.
  3. Write the verdict. A pass/fail attestation is written to our registry on Mantle — any contract can call isAttestedSafe(addr).
  4. Bind to identity. If the target is an ERC-8004 agent, the verdict is also written to its reputation, so safety travels with the agent.
